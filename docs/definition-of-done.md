# Definition of Done

## Descripción
Criterios que debe cumplir un item para ser considerado completo.

## Para Cada Tarea

- [ ] Requisitos entendidos y documentados
- [ ] Solución diseñada
- [ ] Código implementado
- [ ] Self-review completado
- [ ] Tests escritos (>80% cobertura)
- [ ] Tests pasan localmente
- [ ] Documentación actualizada
- [ ] Code review aprobado
- [ ] Integración con rama main exitosa
- [ ] No hay regressions detectadas

## Para Cada Historia de Usuario

- [ ] Criterios de aceptación cumplidos
- [ ] Todas las tareas completadas
- [ ] Tests E2E pasan
- [ ] Documentación de usuario disponible
- [ ] Documentación técnica actualizada
- [ ] Performance aceptable
- [ ] Security review completado
- [ ] Data quality validada
- [ ] Demostración a stakeholders
- [ ] Listo para producción

## Para Cada Sprint

- [ ] Todas las historias en Done
- [ ] Retrospectiva completada
- [ ] Documentación de sprint actualizada
- [ ] Bugs críticos resueltos
- [ ] Deuda técnica bajo control
- [ ] Métricas de sprint registradas

## Para Release

- [ ] Todas las historias de release completadas
- [ ] Validación de regresión completa
- [ ] Performance testing aprobado
- [ ] Security testing aprobado
- [ ] Documentación de release preparada
- [ ] Plan de rollback documentado
- [ ] Equipo notificado
- [ ] Producción lista
